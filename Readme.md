# Gas lift Optimization - Official MSc Degree Implementation - Lucas Kuhnen Costa

This repository is exclusively designed to isolate all useful python files for my MSc Thesis. The focus is on:

### 1) BLACK BOX SIMULATOR
Define a rigorous "black-box" model in CasADi, that allows for flexible values of: $BSW$, $GOR$, $PI$, $K_{gd}$, $K_{inj}$, and $K_{pr}$. This is done in black_box_model.py inside "black box simulator"

### 2) SURROGATE SIMULATOR
Define a surrogate model that will be used to train the PINNs:
   2.1) In CasADi, called "surroghate_model_casadi", this is done for VALIDATION plots of the model difference
   2.2) In Pytorch, called "surrogate_model_torch", which is a safer version, for training will be used to come upm with the "surrogate_simulator"
   2.3) Additionally, here there is a file to check feasibility of randomly generated collocation points: "check_feasibility"

### 3) A SOPHISTICATED SIMULATION ENGINE
This engine is used to make simulations for a different grid, models, etc. It contains:  
   3.1) A "make_model" function that builds a model according to the data that it receives using block_builders utilities  
   3.2) A "run_sweep" function that makes a sophisticated evaluation of the model in a grid, facilitating convergence of this map  
   3.3) A sophisticated combination of "extract_stability_boundary_from_grid" and "fit_boundary_polynomial" to come up with the constraint for the well  
   3.4) TBD PINN simulation engine equivalent to "make_model", so that the results matrix is adequately compared with the others generated by "true" simulators  

### 4) A SOPHISTICATED PLOTTING ENGINE
This engine is used for all plotting tasks. It contains:  
   4.1) A generic and well designed "plot_surface" for response plots  
   4.2) "plot_stability_map", which always work with "overlay_boundary_and_fit" and a "poly_to_string"  
   4.3) "plot_w_o_contour", which coordinates the plotting a heatmap from simulation inputs  

### 5) MODEL STUDIES FILES
These files generate a great deal of the figures for the dissertation and for general understanding of the problem  
   5.1) "black_box_well_study"'s main objective is to help us design different well parametrs for the problem and visualize the black box responses
         Figures here generated include:
            5.1.1) Stability plots  
            5.1.2) Heatmaps  
            5.1.3) The $w_out$ as a function of $u_2$ curve for the thesis or article
            5.1.3) Any response curve that you wish in a client-oriented way  
5.2) "surrogate_well_study" is more simple, its only function is to examine the difference between black box and surrogate
the difference between the surrogate and the rigorous in most important plots  
5.3) "pinn_well_study" is also simple in a way that does the same as the "surrogate_well_study", but using a PINN. The plots are the same  
5.4) FOCUSED STUDIES FOLDER - Folders with unofficial studies, irrelevant for the thesis but important for model building. Will be deleted after the thesis is ready and before the content is published

### 6) OPTIMIZATION FILES
The optimization folder contains simulator engines that will be not likely more changed in the three weeks, as they are generic enough  
6.1) "production_optimizer" is a robust optimizer that uses the model that is available to it
(no matter if rigorous, surrogate or PINN) to optimize field production. 
It generates important information for the thesis and has important characteristics including:  
6.1.1) The value of all constraints in the unconstrained problem, data that is necessary in order to create the constrained problem.
This can be obtained by running it in "unconstrained" mode, which are called in "optimization studies"
6.1.2) Generically implement constraints   
6.1.3) Multi-well model generation and constraint definition (as long as parameters are inputted)   
6.1.4) Precise time measurement for solving the full optimization   
6.1.5) Possibility of solving the NLP with different solver and solver options
